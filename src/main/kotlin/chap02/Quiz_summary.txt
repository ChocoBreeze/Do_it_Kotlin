코틀린은 자료형에 대한 오류를 미리 잡을 수 있는 정적 언어입니다.

NPE ?
Null Pointer Exception으로 값이 할당되지 않아 Null인 상태의 변수에 접근할 때 발생하는 예외

코틀린 프로그램 진입점 역할을 하는 함수는?
main()

코틀린은 멀티패러다임 언어이다. 다음 중 지원하는 패러다임은 무엇인가요?
함수형, 객체지향형

코틀린의 특징
1. 문장 끝에 세미콜론 없어도 된다.
2. nullable타입과 non-null type으로 자료형 선언이 가능하다
3. 다양한 플랫폼에서 동작한다
4. NPE 현상을 제어하기 쉽다.

값 변경이 불가능한 변수 선언 -> val
값 변경이 가능한 변수 선언 -> var

val name : String = 24의 잘못된 점?
String형이므로 문자열로 할당해야 함!

val num = 0x3F는 어떤 값으로 추론되나요?
Int형. (처음에 F만 보고 Float형 고름..)
